AI FINGERPRINT ATTENDANCE SYSTEM WITH REAL-TIME MALPRACTICE DETECTION
1.0 INTRODUCTION
This document presents a comprehensive system that integrates fingerprintbased biometric authentication with realtime AIpowered monitoring for examination integrity. The system addresses two critical needs: (1) ensuring that only registered, eligible students are marked present through robust multifactor authentication, and (2) automatically detecting and alerting invigilators to suspicious behaviors such as unauthorized object usage, multiple faces, absence, excessive head movement, talking, or material exchange.
By embedding stateoftheart deep learning architectures transfer learning for fingerprint recognition, YOLOv12 for object detection, ResNet50V2GRU and CNNLSTM for temporal behavior analysis the system achieves high detection accuracy while respecting privacy through homomorphic encryption of biometric templates. The system is designed for onpremise examination halls with optional cloud extension, covering student enrolment, examination session management, realtime attendance, AIbased proctoring, alerting, logging, and report generation. Hardware procurement (fingerprint scanners, cameras, edge gateways) is outside the software scope.

5.0 Additional Intelligent Features
5.1 Alert System
* Visual alerts: A red border or popup on the invigilator dashboard, identifying the camera, timestamp, and anomaly type.
* Audio alerts: A configurable system beep (or texttospeech message) to draw immediate attention.
* Push notifications: Optional integration with mobile app or email for remote supervisors.
* Alert aggregation: To avoid alert fatigue, similar events are grouped (e.g., 5 movement alerts within 10 seconds become one aggregated alert).

5.3 Screenshot and Video Capture (Optional)
Upon crossing the suspicion threshold, the system captures:
i. A highresolution JPEG frame (configurable: 1–5 frames).
ii. A short video clip (e.g., 10 seconds before and after the event) in MP4 format.
Media files are encrypted using AES256 and stored with access restricted to authorized personnel.
5.4 Scalability and Deployment
The system is designed as a cloudnative microservices architecture with the following scalability features:
i. Horizontal scaling: Multiple AI proctoring service instances can process video streams from different examination halls concurrently.
ii. Load balancing: Incoming verification requests are distributed across available fingerprint matching nodes.
iii. Elasticity: Autoscaling groups adjust resources based on exam schedules (peak vs. offpeak).
iv. Edgecloud hybrid: Fingerprint matching can be performed onpremise (edge gateway) for low latency, while AI proctoring runs in the cloud or on dedicated GPU clusters.
v. Future extensions: Integration with central examination boards, multidepartment deployments, and support for remote proctoring.

8.0 System Architecture and Data Flow
8.1 FiveLayer CloudNative Architecture
The system follows a fivelayer cloudnative architecture:
i. Frontend Layer – Web dashboards (React/Vue.js) for administrators, examination officers, and invigilators. Realtime updates via WebSockets.
ii. Application Layer – Microservices (Node.js/Spring Boot) for student management, exam session setup, attendance recordkeeping, and report generation.
iii. Biometric Processing Layer – Edge fingerprint scanners (e.g., SecuGen Hamster Pro) with local template caching; matching engine using LMAES homomorphic comparison.
iv. AI Proctoring Layer – YOLOv12 for object detection, ResNet50V2GRU for sequential behavior, CNNLSTM for anomaly scoring, and face detection modules (RetinaFace). Deployed on GPU instances.
v. Data Layer – Encrypted storage (LMAES ciphertexts in PostgreSQL), Redis for session caching, RabbitMQ/Kafka for alert streaming, and S3compatible object storage for evidence media.
8.2 Data Flow
i. Enrolment phase: Admin registers student ? fingerprint scanner captures image ? minutiae extracted ? template encrypted via LMAES ? stored in database.
ii. Examination phase:
a. Student scans fingerprint at hall entrance ? edge gateway performs encrypted match ? attendance recorded (timestamp, hall, session).
b. Cameras stream video to AI proctoring service ? YOLOv12 detects objects ? ResNet50V2GRU/CNNLSTM analyses behavior ? alerts generated if threshold exceeded.
c. Alerts are pushed via message queue to WebSocket server ? invigilator dashboard updates in real time.
iii. Postexam phase: Officer generates attendance and malpractice reports (PDF/Excel) from the data layer.

10.0 Conclusion
The proposed system provides a reliable, efficient, and scalable solution for examination management. By combining fingerprint biometrics (with transfer learning achieving 100% accuracy) with AIbased monitoring (YOLOv12 at 97.15%, ResNet50V2GRU at F1=1.0, and CNNLSTM at 87.8%), the system significantly reduces impersonation and malpractice. Homomorphic encryption (LMAES, Kyber AHE) ensures biometric privacy without sacrificing performance. The fivelayer cloudnative architecture supports concurrent largescale examinations, realtime alerting, comprehensive logging, and evidence capture. This system restores trust in academic assessment processes while meeting modern security, scalability, and compliance demands.
